# Migration: 0.32.0 to 0.34.0

## Summary

Replace `fab/current` pointer file with `.fab-status.yaml` symlink at repo root. Add `id` field to all `.status.yaml` files. Update `.gitignore`.

## Pre-check

1. Confirm `fab/.kit/VERSION` reads `0.34.0` or higher (kit has been upgraded)
2. Check if `.fab-status.yaml` already exists at repo root — if so, migration is already done (skip symlink step)

## Changes

### 1. Convert `fab/current` to `.fab-status.yaml` symlink

If `fab/current` exists and is non-empty:
1. Read the file contents. It may be a 2-line file (ID on line 1, folder name on line 2) or a 1-line file (just the folder name or ID)
2. Extract the folder name:
   - If 2+ lines: use line 2 (trimmed)
   - If 1 line: the value is either a 4-char ID or a folder name. If it's 4 chars, scan `fab/changes/` for a matching folder (pattern `??????-{id}-*`). If it's longer, use it as the folder name directly
3. Verify the change folder exists at `fab/changes/{folder}/`
4. Create the symlink: `ln -s "fab/changes/{folder}/.status.yaml" .fab-status.yaml`
5. Remove `fab/current`: `rm fab/current`

If `fab/current` is empty or absent: skip this step (no active change to migrate).

### 2. Backfill `id` field in existing `.status.yaml` files

For each `.status.yaml` in `fab/changes/` (including `fab/changes/archive/` recursively):
1. Check if the file already has an `id:` field at the top — if so, skip
2. Extract the 4-char change ID from the folder name (second hyphen-separated component of the YYMMDD-XXXX-slug format)
3. Add `id: {extracted_id}` as the first line of the file (before the existing `name:` line)

### 3. Update `.gitignore`

1. If `.gitignore` contains `fab/current`, replace it with `.fab-status.yaml`
2. If `.gitignore` does not contain `fab/current` but also doesn't contain `.fab-status.yaml`, add `.fab-status.yaml` to the file

## Verification

1. `fab/current` should not exist
2. `.fab-status.yaml` should be a symlink (if there was an active change) — verify with `ls -la .fab-status.yaml`
3. The symlink target should be `fab/changes/{name}/.status.yaml` — verify with `readlink .fab-status.yaml`
4. All `.status.yaml` files should have an `id:` field as their first line
5. `.gitignore` should contain `.fab-status.yaml` and not `fab/current`
6. Run `fab preflight` to confirm resolution still works
