# Migration: 0.2.0 to 0.3.0

## Summary

Memory and specs directories have moved from `fab/` to `docs/`. This migration relocates `fab/memory/` → `docs/memory/` and `fab/specs/` → `docs/specs/`, and updates `fab/constitution.md` to reference the new paths.

## Pre-check

- `fab/memory/` directory exists
- `fab/specs/` directory exists
- `docs/` directory does NOT already exist (or does not contain `memory/` or `specs/`)

## Changes

1. **Move directories**:
   ```bash
   mkdir -p docs
   mv fab/memory docs/memory
   mv fab/specs docs/specs
   ```

2. **Update `fab/constitution.md`** — replace path references:
   - `fab/memory/` → `docs/memory/`
   - `fab/specs/` → `docs/specs/`

3. **Update `README.md`** (if it exists) — replace path references:
   - `fab/memory/` → `docs/memory/`
   - `fab/specs/` → `docs/specs/`

4. **Update path references in relocated files** — within `docs/memory/` and `docs/specs/`, replace any remaining references:
   - `fab/memory/` → `docs/memory/`
   - `fab/specs/` → `docs/specs/`

## Verification

- `docs/memory/index.md` exists and is readable
- `docs/specs/index.md` exists and is readable
- `fab/memory/` no longer exists
- `fab/specs/` no longer exists
- `grep -r 'fab/memory/\|fab/specs/' fab/constitution.md` returns no matches
- Cross-links between `docs/memory/` and `docs/specs/` resolve correctly (e.g., `../specs/index.md` from `docs/memory/index.md`)
