# Migration: 0.3.0 to 0.4.0

## Summary

The first pipeline stage has been renamed from `brief` to `intake`. This migration renames `brief.md` → `intake.md` in active change folders, updates `.status.yaml` stage keys across all change folders (including archive), and updates internal content references.

## Pre-check

- At least one directory under `fab/changes/` contains `brief.md` or has `brief:` in `.status.yaml`

## Changes

### 1. Rename artifact files

For each **active** change folder in `fab/changes/` (excluding `archive/`):

- If `brief.md` exists, rename it to `intake.md`
- In the renamed `intake.md`, replace the `# Brief:` heading prefix with `# Intake:`
- In the file content, replace references to the `brief` stage/artifact name with `intake` (e.g., "brief writing" → "intake writing"). Do NOT replace the English adjective "brief" (meaning short/concise) or occurrences where "brief" appears as part of a change slug or proper name.

### 2. Update `.status.yaml` stage keys

For **all** change folders in `fab/changes/` (including `archive/`):

- In `progress:`, replace `brief:` key with `intake:` (preserving the value)
- In `stage_metrics:`, replace `brief:` key with `intake:` (preserving the value)

### 3. Archive folders — artifact files unchanged

Existing `brief.md` files and their content in `fab/changes/archive/` are historical artifacts. Do NOT rename them or update their headings/content. Only `.status.yaml` stage keys are migrated (step 2).

## Verification

- `find fab/changes -maxdepth 2 -name brief.md -not -path '*/archive/*'` returns no matches
- `grep -r 'brief:' fab/changes/*/.status.yaml fab/changes/archive/*/.status.yaml` returns no matches for stage keys (may still match in `name:` fields for change slugs containing "brief")
- Each migrated `intake.md` (active changes) starts with `# Intake:`
- Every `.status.yaml` (active and archive) has `intake:` in both `progress:` and `stage_metrics:` (where applicable)
